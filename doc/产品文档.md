## 一、产品概述

Simu 是一款提供虚拟号码接收短信验证码的服务平台。用户可通过 iOS 应用购买来自全球 100+ 国家的虚拟号码，用于注册各类互联网平台账号，保护个人隐私。

### 核心价值
- 🌍 **全球覆盖**: 支持 100+ 国家、多种服务平台的虚拟号码
- 🔒 **隐私保护**: 使用虚拟号码代替真实手机号
- ⚡ **即时接收**: 激活后自动轮询获取验证码
- 💰 **积分体系**: 按次计费，支持会员等级折扣

### 3.4 订单管理

| 功能 | 端点 | 方法 | 说明 |
|------|------|------|------|
| 获取订单列表 | `/orders` | GET | 支持按状态筛选 |
| 创建订单 | `/orders` 或 `/orders/create` | POST | 扣除积分，状态 pending |
| 激活订单 | `/orders/{id}/activate` | POST | 获取号码，20分钟倒计时 |

| 订单详情 | `/orders/{id}` | GET | 含服务/国家信息 |

#### 订单状态流转

```
pending (待使用)
  ├── 激活 → active (等待短信) → 收到短信 → completed (已完成)
  ├── 激活 → active → 20分钟超时 → completed (号码使用已过期)
  ├── 取消 → cancelled (已取消，退积分)
  └── 24小时未激活 → expired (已过期，不退积分)
```

#### 20分钟倒计时机制
1. 激活订单时设置 `expires_at = NOW() + 20 MINUTE`
2. 前端显示实时倒计时（MM:SS 格式）
3. 剩余 5 分钟时倒计时变红色警告
4. Cron 任务检测超时，标记为 `completed`（不退积分）
5. 超时后自动调用 Hero-SMS 取消号码

#### 激活前引导弹窗
- 步骤 1: 复制激活后的电话号码
- 步骤 2: 前往目标服务平台注册并发送验证码
- 步骤 3: 返回应用查看验证码
- 警告提示: 20 分钟内未收到短信订单将标记为完成

## 五、定时任务

### 5.1 短信轮询 (fetch-sms.php)

**执行频率**: 每 2-5 秒一次

**功能**:
1. 查询所有 `active` 状态的订单
2. 检查是否超时（`expires_at` < 当前时间）
   - 超时 → 标记为 `completed`（号码使用已过期，不退积分）
   - 调用 Hero-SMS 取消号码
3. 未超时 → 调用 Hero-SMS 获取短信
   - 收到短信 → 提取验证码 → 保存数据库 → 标记订单为 `completed`
   - 调用 Hero-SMS 完成激活（停止计费）
4. 检查 `pending` 订单是否超过 24 小时未激活
   - 超时 → 标记为 `expired`（不退积分）
   - 发送过期通知

**Cron 配置**:
```bash
* * * * * php /path/to/backend/cron/fetch-sms.php >> /var/log/sms-receiver/cron.log 2>&1
```

## 六、安全机制

### 6.1 认证安全
- JWT Token 验证（7天有效期）
- 设备 ID 绑定
- API Key 验证（`X-API-Key` 请求头）
- 密码 BCrypt 哈希存储

### 6.2 数据安全
- CORS 配置（生产环境限制来源）
- SQL 参数化查询（防注入）
- 敏感信息不返回前端（如 password_hash）
- Hero-SMS API Key 从数据库安全读取

### 6.3 业务安全
- 订单所有权验证（用户只能操作自己的订单）
- 积分余额校验（防止负余额）
- 事务处理（保证数据一致性）
- 退款逻辑保护 `total_spent`（维持会员等级）

### 6.4 公开端点
以下端点不需要 API Key 验证：
- `/health`
- `/banners`
- `/settings`
- `/auth/password-login`
- `/auth/manual-register`
- `/auth/forgot-password`
- `/auth/change-password` (替换 reset-password 流程)
- `/auth/verify-password`

> ⚠️ **v1.2.0 重大变更**: `/auth/reset-password` 流程**已废弃**。新流程只需要 `/auth/forgot-password` 一次调用，后端直接返回新密码，前端展示给用户。

## 九、Hero-SMS 集成

### 9.1 对接流程

1. **获取号码**: 调用 `getNumber(service, country)` 获取虚拟号码
2. **等待短信**: 轮询 `getSMS(hero_order_id)` 获取短信内容
3. **完成激活**: 收到短信后调用 `complete(hero_order_id)` 停止计费
4. **取消号码**: 超时或用户取消时调用 `cancelNumber(hero_order_id)`

### 9.2 错误处理

- 获取号码失败 → 订单标记为 `failed`，自动退款
- 获取短信失败 → 返回 null，继续轮询
- 完成激活失败 → 记录日志，不影响订单状态

### 9.3 库存查询

- 调用 `getStock(service, country)` 查询可用号码数量
- 结果缓存 60 秒，减少 API 调用

## 十一、业务规则

### 11.1 积分规则
- 购买号码扣除积分
- 充值获得积分
- 首充双倍积分
- pending 订单取消退还积分
- active 订单取消/超时不退还积分

### 11.2 订单规则
- 创建订单后 24 小时内必须激活
- 激活后 20 分钟内必须收到短信
- 超时未激活 → expired（不退积分）
- 超时未收到短信 → completed（不退积分，号码使用已过期）

### 11.3 会员规则
- 按累计消费金额计算等级
- 退款不扣除 `total_spent`
- 等级越高折扣越大
- 首充用户享受更优价格系数

## 十三、常见问题

### Q1: 为什么激活后只有 20 分钟？
A: Hero-SMS 供应商规定号码等待验证码的时间最长 20 分钟，超时后号码将被回收。

### Q2: 订单超时后积分能退吗？
A: 不能。号码已分配给用户使用，即使未收到短信，成本已产生。

### Q3: pending 订单可以取消吗？
A: 可以。未激活的订单取消后全额退还积分。

### Q4: 如何成为会员？
A: 累计消费达到对应金额即可自动升级，无需手动申请。

### Q5: 首充奖励如何获得？
A: 注册后 24 小时内完成首次充值即可获得双倍积分。
