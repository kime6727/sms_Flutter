### 📌 工作原理

```
PHP 代码发起查询
      ↓
MySQL 执行 SQL
      ↓
有索引 → 快速定位数据（B-Tree 查找）
无索引 → 全表扫描（逐行比对）
```

### 🔍 举例说明

**没有索引时**（全表扫描）：
```sql
-- 查询某个用户的待使用订单
SELECT * FROM orders WHERE user_id = 5 AND status = 'pending';
-- 需要扫描全部 6 条记录 → 未来可能是 10 万条
```

**有索引后**（B-Tree 查找）：
```sql
-- 添加索引
CREATE INDEX idx_orders_user_status ON orders(user_id, status);
-- 直接定位到目标记录 → 只需读取 1 条
```

### 💡 总结

- **索引是 MySQL 的功能**，PHP 只是执行 SQL 查询
- 添加索引后，PHP 代码**不需要任何修改**
- 效果立竿见影，特别是数据量大时
