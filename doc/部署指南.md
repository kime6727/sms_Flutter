# 生产环境部署指南

> 目标：在云服务器（dokploy / VPS）上部署 SMS 接码平台后端，
> 正确配置 TiDB Cloud 数据库连接，修复 `数据库连接失败` 错误。

---

## 1. 现象与根因

### 1.1 错误信息

```
Fatal error: Uncaught Exception: 数据库连接失败:
SQLSTATE[HY000] [2002] No such file or directory
in /app/lib/Database.php:76
```

`SQLSTATE[HY000] [2002] No such file or directory` 表示 PHP 在尝试连接
**本地 Unix 套接字**（`/var/run/mysqld/mysqld.sock` 等），而不是走 TCP 远程连接。

### 1.2 根因

`backend/config/database.php` 读取的是 `backend/.env` 中的 `DB_HOST/DB_PORT/...`，
当 `.env` 文件不存在时，会回落到默认值：

```php
$dbHost = $env['DB_HOST'] ?? 'localhost';   // ← 这里
$dbPort = $env['DB_PORT'] ?? 3306;
$dbName = $env['DB_NAME'] ?? 'sms_receiver';
$dbUser = $env['DB_USER'] ?? 'root';
$dbPass = $env['DB_PASS'] ?? '';
```

`localhost` 在 PDO 中默认走 socket 连接，所以才会出现 `No such file or directory`。

### 1.3 修复方案

在服务器上的 `backend/.env` 写入 **TiDB Cloud** 的真实连接信息（含 SSL 配置）。

---

## 2. 在 TiDB Cloud 上准备数据库

> ⚠️ TiDB Cloud SQL Editor **不允许 CREATE DATABASE**，必须先在 UI 建库。

### 2.1 创建 `sms_receiver` 数据库

1. 登录 [TiDB Cloud](https://tidbcloud.com/)
2. 进入目标 Cluster → 左侧 **SQL Editor**
3. 顶部 database 下拉框 → **Create database**
4. 名称填 `sms_receiver`，点击创建
5. 顶部 database 下拉框切换到 `sms_receiver`

### 2.2 执行初始化 SQL

把 `backend/database.sql` 整个文件内容复制到 SQL Editor → 点击 **Run**。

成功后会显示 21+ 张基础表已创建，含初始管理员账号 `admin/admin123`。

### 2.3 获取连接信息

右上角 **Connect** → **Standard Connection** → 看到：

```
HOST:  gateway01.ap-southeast-1.prod.alicloud.tidbcloud.com
PORT:  4000
USERNAME:  xxxxxxxxxx.root
PASSWORD:  xxxxxxxxxx
CA:  /etc/ssl/cert.pem
```

把这一份信息保留好，下一步会用到。

## 3. 在服务器上配置 `.env`

> 假定项目根目录是 `/app`（dokploy 默认），后端代码在 `/app/backend`。
> 如使用其他路径，请对应替换。

### 3.1 复制模板

```bash
cd /app/backend
cp .env.example .env
```

### 3.2 编辑 `.env` 填入真实值

```bash
nano .env   # 或 vi / vim
```

将以下字段替换为 TiDB Cloud 真实信息：

```ini
# --- 数据库 ---
DB_HOST=gateway01.ap-southeast-1.prod.alicloud.tidbcloud.com
DB_PORT=4000
DB_NAME=sms_receiver
DB_USER=2tNw6XTWxveXcVU.root
DB_PASS=81524yPR2teDEIH7

# --- SSL（TiDB Cloud 强制）---
DB_SSL_ENABLED=1
DB_SSL_CA=/etc/ssl/cert.pem
DB_SSL_VERIFY=true

# --- API ---
# 生成: openssl rand -hex 32
API_KEY=<your_random_api_key>
HEROSMS_API_KEY=<your_herosms_key>

# --- 认证密钥 ---
# 生成: openssl rand -hex 32
AUTH_SECRET_KEY=<your_auth_secret>

# --- 应用 ---
APP_URL=https://sms.niceapp.eu.cc
APP_ENV=production
CORS_ALLOWED_ORIGINS=https://sms.niceapp.eu.cc
```

### 3.3 关键字段说明

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `DB_HOST` | ✅ | TiDB Cloud 网关地址，**不能**是 `localhost` |
| `DB_PORT` | ✅ | TiDB Cloud 默认 `4000`，本地 MySQL 用 `3306` |
| `DB_NAME` | ✅ | 在 TiDB Cloud UI 创建的真实库名（不能是 `sys`） |
| `DB_SSL_ENABLED` | ✅ | TiDB Cloud 强制设为 `1` |
| `DB_SSL_CA` | ✅ | 服务器上 CA 证书路径，docker 镜像内置 `/etc/ssl/cert.pem` |
| `APP_URL` | ✅ | 必须是公网可访问的 HTTPS，用于 hero-sms webhook 回传 |
| `APP_ENV` | ✅ | 生产环境必须 `production` |
| `CORS_ALLOWED_ORIGINS` | ✅ | 禁止 `*`，填前端域名 |

### 3.4 验证 SSL 证书存在

```bash
ls -lh /etc/ssl/cert.pem
# 期望: -rw-r--r-- 1 root root ... /etc/ssl/cert.pem
```

如果不存在，dokploy 镜像通常自带 Mozilla CA bundle；如真没有，临时设
`DB_SSL_VERIFY=false`（仅调试用），同时启用 `DB_SSL_CA_CONTENT` 注入证书内容。

## 4. 重启服务并验证

### 4.1 重启 PHP-FPM / 反向代理

dokploy 上点击 **Redeploy**，或手动：

```bash
docker restart <php_container>
# 或
systemctl restart php8.2-fpm nginx
```

### 4.2 健康检查

```bash
curl -sS https://sms.niceapp.eu.cc/health
```

期望输出：

```json
{ "status": "ok", "timestamp": "2026-06-03 ..." }
```

### 4.3 简单接口冒烟

```bash
# 公开接口（无需 API Key）
curl -sS https://sms.niceapp.eu.cc/banners | head -c 200
curl -sS https://sms.niceapp.eu.cc/settings | head -c 200
```

### 4.4 受保护接口

```bash
curl -sS -H "X-API-Key: <your_api_key>" \
  https://sms.niceapp.eu.cc/services | head -c 200
```

如果 `X-API-Key` 不匹配会返回 `403 API Key 无效或缺失`，匹配则返回服务列表。

## 5. 常见问题排查

| 现象 | 排查点 |
| --- | --- |
| `SQLSTATE[HY000] [2002] No such file or directory` | `.env` 不存在或 `DB_HOST=localhost`，走 socket |
| `SQLSTATE[HY000] [2002] Connection refused` | `DB_HOST/DB_PORT` 写错，或服务器无法访问 TiDB Cloud（防火墙 / 区域不匹配） |
| `SQLSTATE[HY000] [1045] Access denied` | `DB_USER/DB_PASS` 错误 |
| `SQLSTATE[HY000] [1049] Unknown database 'sms_receiver'` | 未在 TiDB Cloud UI 创建 `sms_receiver` 库 |
| `SSL operation failed` | `DB_SSL_ENABLED=1` 但 `DB_SSL_CA` 路径错；检查 `ls -lh /etc/ssl/cert.pem` |
| `CORS 跨域错误` | `CORS_ALLOWED_ORIGINS` 必须包含前端域名（多个用逗号分隔） |

## 6. 安全清单

- [x] `.env` 在 `.gitignore` 中，**绝不**提交
- [x] `API_KEY` 与 `AUTH_SECRET_KEY` 用 `openssl rand -hex 32` 生成
- [x] TiDB Cloud 数据库用户密码保存在 TiDB Cloud Secrets，不要同步到任何聊天工具
- [x] 生产环境 `APP_ENV=production`，自动隐藏 PHP 错误详情
- [x] `CORS_ALLOWED_ORIGINS` 禁止 `*`
- [x] 定期在 TiDB Cloud 控制台轮换 root 密码
